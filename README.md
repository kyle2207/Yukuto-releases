# 悠投 Yukuto · 使用手冊

[![.NET](https://img.shields.io/badge/.NET-8.0-blue)](https://dotnet.microsoft.com/)

**悠投 (Yukuto)** 是台灣股東自行完成股東會電子投票的個人輔助工具。整合集保「股東 e 票通」FidO 登入，協助你一鍵對名下持股全部投贊成、自動截圖留存，省去每場手動操作的麻煩。

> 🔄 **免維護、自動更新**：安裝一次即可。每次啟動會自動偵測新版本，有更新時跳出提示，一鍵確認即保持最新版 —— 集保平台改版、功能優化、問題修正都自動推送，無需手動下載。
>
> ⚠️ 悠投依賴集保電子投票平台與股代網運作。若上游平台改版，悠投可能暫時無法使用，需等待修正更新後恢復；此期間請以人工方式完成投票，造成不便敬請見諒。

> 📥 想離線閱讀或列印？[**下載 PDF 版手冊**](https://github.com/kyle2207/Yukuto-releases/raw/main/docs/manual/user-manual.pdf)　·　截圖中的個人資料（身分證字號、姓名等）均已做模糊處理。

---

> 🪪 **使用前必備｜行動自然人憑證（TW FidO）**
>
> 悠投以集保「股東 e 票通」FidO 登入，**必須先在手機申辦並綁定「行動自然人憑證」才能操作**。這是由**內政部**推出的服務，把自然人憑證放進手機，免插卡、免帳號密碼，靠手機生物辨識（指紋／臉部）即可驗證。**尚未申辦者，請先完成綁定再開始使用。**

先在手機安裝「**行動自然人憑證 TW FidO**」APP（App Store／Google Play），再擇一方式綁定：

| 申辦方式 | 說明 |
|----------|------|
| **網頁 QR Code 綁定** | 電腦以**自然人憑證 IC 卡＋讀卡機**登入官網產生 QR Code，用手機 APP 掃描完成綁定 |
| **APP NFC 靠卡綁定** | 手機 APP 直接靠實體自然人憑證卡（卡號 TP07 開頭之雙介面卡）完成綁定 |
| **臨櫃申辦** | 攜帶**身分證**、email 與手機，至全國任一**戶政事務所**辦理 |

📘 三種方式的詳細步驟皆於官方申辦教學頁：https://fido.moi.gov.tw/pt/main/teaching　·　☎️ 內政部客服 **0800-080-117**

## 1. 安裝與系統需求

### 下載與安裝

1. 前往 [Releases](https://github.com/Kyle2207/Yukuto-releases/releases) 頁面，下載最新版的 **`Yukuto-win-Setup.exe`**。
2. 執行安裝程式（**不需系統管理員權限**），程式會自動安裝到 `%LocalAppData%\Yukuto\`。
3. 從開始功能表或桌面捷徑啟動 **悠投**。

### 系統需求與套件

| 項目 | 需求 |
|------|------|
| 作業系統 | Windows 10 (1809+) / Windows 11，**x64** |
| **Google Chrome** | **必須安裝**（建議 120 以上版本）—— 悠投靠它驅動投票流程 |
| **行動自然人憑證（TW FidO）** | **必須** —— 集保登入身分驗證用，需先申辦並在手機綁定（見手冊開頭重點說明） |
| .NET Runtime | **不需要**，安裝程式已內含執行環境 |
| 其他套件 | 無，唯一外部需求就是 Google Chrome |

> 💡 若未安裝 Chrome，悠投首次啟動會偵測並提示一鍵安裝。

### 自動更新

悠投採 **OTA 自動更新**，安裝一次後即免手動維護：

- 每次啟動會在背景靜默檢查新版本，有更新時跳出提示，**一鍵確認即套用**。
- 集保平台改版、功能優化、問題修正都透過自動更新推送，**無需自行重新下載安裝**。
- 若遇集保平台重大改版導致暫時無法使用，請稍候修正版更新；期間請以人工方式投票。

## 2. 開始使用

本章涵蓋從第一次啟動到完成投票的完整流程。

### 2.1 首次啟動：使用條款與隱私政策

**第一次啟動**（或條款更新時），會先跳出「使用條款與隱私政策」視窗，須閱讀並同意才能繼續使用。

![使用條款與隱私政策](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/dialog-terms.png)

內容涵蓋六個部分：

1. **工具性質與使用範圍** —— 電子投票輔助軟體，與集保公司、櫃買中心等官方機構無隸屬或授權關係。
2. **使用限制與使用者責任** —— 僅就本人合法持有、或經帳戶持有人合法授權之帳戶行使投票權；自行確保合法權源、遵循集保平台條款與法令，風險自負。
3. **委託代領／代售服務** —— 紀念品代領、代售為獨立委託關係（限實體紀念品、非證券交易）；委託人保證合法權源，服務費依計價。
4. **免責聲明** —— 投票決策自負；工具以「現狀」提供、不保證持續可用。
5. **個人資料蒐集與使用** —— 蒐集帳號信箱與投票截圖（截圖含投票人姓名、股號、開會資訊，採權限控管、保留 1 年後自動清除）；集保身分證字號僅於本機驗證、不上傳。資料不出售、不提供第三方廣告。
6. **條款變更** —— 條款可能更新，重大變更會於更新說明中告知。

| 按鈕 | 作用 |
|------|------|
| **我已閱讀並同意，繼續使用** | 同意條款並進入程式 |
| **不同意，關閉程式** | 不同意則關閉程式、無法使用 |

> 💡 同意後就不會再跳出；只有**條款改版**時才會再次出現請你確認。

### 2.2 登入悠投帳號

使用悠投需要一個**平台帳號**，以你的 **Google 帳號**或 **Facebook 帳號**登入即可。

> 📝 **不需另外註冊**：第一次以 Google／Facebook 登入時，系統會自動為你建立平台帳號；日後用同一個帳號登入即可。

開啟程式後，按畫面指示以 Google 或 Facebook 登入，登入後即進入下方的投票主畫面。

![登入畫面](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/login-1-account.png)

| 欄位 | 說明 |
|------|------|
| **集保帳號** | 輸入你的身分證字號（作為集保平台登入帳號） |
| **記住帳號** | 勾選後下次自動帶入集保帳號 |
| **登入方式** | 行動自然人憑證（TAIWAN FidO） |

> 🔒 身分證字號僅於本機記憶體暫存供當次登入，不會儲存或上傳。

### 2.3 集保 FidO 掃碼登入

輸入集保帳號後按「**登入並載入清單**」，下方會出現 QR Code。請用手機的 **「行動自然人憑證 APP」** 掃描完成驗證。

![FidO 掃碼](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/login-2-qr.png)

> 📱 使用前手機需先安裝「行動自然人憑證 APP」並完成註冊。
> ⏱️ QR Code 有效時間約 60 秒，逾時可按「更新 QR Code」重新取得。
> 完成手機端驗證後，此區會自動收起並載入你可投票的股東會清單。

### 2.4 自動投票

登入成功後，工具會載入你名下所有股東會清單。頂部會顯示「可投票股東會 N 筆（未投票／已投票／已截止）」的概況。

![投票清單](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/voting-list.png)

#### 搜尋與篩選

清單上方提供搜尋與篩選，方便在大量場次中快速定位：

| 工具 | 說明 |
|------|------|
| **搜尋** | 輸入股票代號或公司名稱，即時過濾清單 |
| **紀念品** | 全部／有紀念品／無紀念品／可代領 |
| **狀態** | 全部／未投票／已投票／已截止 |
| **清除** | 清除搜尋與篩選條件，回到完整清單 |

> 💡「全選未投票」與「開始自動投票」會作用於**目前篩選範圍內**的項目。

#### 操作步驟

1. 於「股東會清單」勾選要投票的場次（或按「**全選未投票**」）。
2. 按「**開始自動投票**」。工具會自動對所有議案投「**贊成**」、選舉議案自動全選並平均分配票數，並處理 reCAPTCHA。
3. 完成後自動擷取**投票證明截圖**並上傳雲端（可於「紀念品總覽」查閱）。

其他按鈕：**匯出清單**（匯出股東會清單）、**重新上傳截圖**（重新擷取投票證明）。

> ⚠️ 預設一律投「贊成」屬技術行為、非投資建議。若不同意「全數贊成」策略，請勿使用本工具投票。

## 3. 帳號設定

從左側導覽列的 **👤（帳號設定）** 圖示進入。管理你的登入帳號與個人資料。

![帳號設定](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/account-blur.png)

### 基本資料

| 欄位 | 說明 | 可否修改 |
|------|------|---------|
| **Email** | 你登入所用的信箱（由 Google／Facebook 提供），作為帳號識別 | 否 |
| **顯示名稱** | 在系統中顯示的名稱 | 由 Google／Facebook 管理，須至該帳戶修改 |
| **帳號建立** | 此帳號的建立日期 | 否 |
| **使用者 ID** | 系統內部識別碼，客服核對身分時可能用到 | 否 |

### 登入方式

顯示此帳號**目前實際綁定**的登入方式（只列出在用的），例如 **Google 帳號**或 **Facebook 帳號**。

> 🔒 只有當帳號**同時綁定兩種以上**登入方式時，才會出現「解除綁定」按鈕 —— 這是為了避免解除掉唯一的登入方式而導致無法再登入。

## 4. 紀念品總覽

從左側導覽列的 **🎁（紀念品總覽）** 圖示進入。此頁以**紀念品資料庫為基準**、聚焦**今年度**，彙整你所有的投票證明與紀念品狀態，並標示**哪些有紀念品的場次你還沒投到**，一眼看出漏掉、沒買到的紀念品。

![紀念品總覽](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/raw-overview.png)

### 上方工具列

| 項目 | 說明 |
|------|------|
| 統計摘要 | 顯示今年度、有投票幾場、⚠️ 有紀念品未投到幾檔、服務費合計／售出／應收金額 |
| **重新整理** | 重新從雲端載入最新明細 |
| **匯出 Excel** | 將明細匯出成 Excel 檔 |

### 投票狀態：一眼看出漏掉的紀念品

本頁以**今年度有紀念品的場次**為基準，用 **投票狀態** 欄區分兩種：

| 投票狀態 | 意義 |
|------|------|
| **有投票**（綠底） | 你已投票，會拿到紀念品 |
| **未投票**（橘底） | ⚠️ 該場**有紀念品但你沒投到**（漏掉、沒買到的） |

> 若你**投了票、但紀念品資料庫沒有該場**，紀念品欄會顯示「（無紀念品）」。
> **未投票**的場次因為沒投到，**不計費**、也**無法設定處置**。

### 分組方式：年度 → 人頭戶

資料先依**年度**摺疊，年度下再依**人頭戶（戶名）**分組：

- 戶名來自投票時系統擷取的股東姓名；**未取得戶名者**歸「**（未標記）**」。
- **未投票**的場次因沒有投票、未取得戶名，一律歸「**（未標記）**」，靠「投票狀態」欄或篩選辨識。
- 若你代多位自然人投票，可在「**人頭戶名單**」登錄正確姓名，明細就會依人頭戶正確歸類（詳見第 5 章）。

### 分組層的操作

每個年度／人頭戶分組的標題列上有對應的操作按鈕：

| 位置 | 按鈕／工具 | 說明 |
|------|-----------|------|
| **年度標題列** | **💰 明細** | 開啟該年度的「服務費明細」視窗：上方卡片為年度總計（一般服務費／售後服務費／售出金額／▶ 應收合計），下方表格列出**各人頭戶**的場次／服務費／售出金額／應收，方便逐一人頭戶對帳統計 |
| **人頭戶標題列** | **批次處置下拉 ＋ ⚡ 套用（N）** | 選一種處置方式後按「套用」，把目前篩選後可見的所有「**待定**」場次一次設為該處置（N＝可套用筆數）。**未投票場次不可套用，篩選未投票時下拉與按鈕會反灰** |
| **人頭戶標題列** | **↪ 歸到** | 僅在戶名未對應到名單時出現；把此戶名歸併到「人頭戶名單」中正確的人（詳見第 5 章） |

### 篩選（每個分組各自提供）

> 💡 **「處置」是你自己的設定，「狀態」是管理員端的回應** —— 兩者不同：你用**處置**表達想怎麼處理紀念品（你的意願）；**狀態**則反映管理員實際收件、領取、出貨、售出的進度（對方的回填）。

| 下拉 | 由誰決定 | 選項 |
|------|---------|------|
| **投票** | 系統 | 全部／有投票／未投票 |
| **處置** | **使用者自己設定**（紀念品處置意願） | 全部／待定／委託代領／委託販售／贈送／不領取 |
| **狀態** | **管理員回填**（實際處理進度的回應） | 全部／待領取／已領取／重複領取／已出貨／已售出／不送出 |
| **繳交** | 系統 | 全部／需繳交／免繳交／未公告 |
| **類別** | 系統依資料動態產生 | 依該分組出現的紀念品類別動態列出 |

### 表格欄位

| 欄位 | 說明 |
|------|------|
| 股票代號／股票 | 公司代號與名稱 |
| **投票狀態** | 有投票（綠）／未投票（橘），見上方說明 |
| 開會日期 | 股東會日期 |
| **股東會性質** | 常會／臨時會 |
| **紀念品／類別** | 該場紀念品說明與分類。**此欄空白／「（無紀念品）」＝該場沒有紀念品** |
| 繳交 | 領取紀念品是否需繳交證件（需繳交／免繳交／未公告） |
| 代領截止 | 紀念品代領截止日（近期會提示剩餘天數，如「07/15」「剩 7 天」） |
| 送出狀態 | 管理員的處理進度（待領取／已領取／已售出…） |
| **紀念品處置** | **可由你修改**的下拉，見下方說明；**未投票場次不可設定** |
| 服務費／售出金額 | 依計價規則計算的費用 |
| 上傳時間 | 投票證明截圖上傳時間 |

### 「紀念品處置」下拉的各選項意義

這是你對該場紀念品的處置意願，**可隨時修改**，並影響是否計費：

| 選項 | 意義 | 是否計費 |
|------|------|---------|
| **待定** | 尚未決定（預設值）；若維持待定，需聯繫管理員並委託其處理 | 暫不計費 |
| **委託代領** | 請平台代為領取紀念品 | 收代領服務費 |
| **委託販售** | 請平台代為轉售紀念品 | 售出後依售出金額計費 |
| **贈送** | 將紀念品贈送（不取回） | 不計費 |
| **不領取** | 放棄該紀念品 | 不計費 |

> 平台不收取證件，僅代領**不需繳交證件**之紀念品；特殊需求請聯繫管理員。

## 5. 人頭戶名單

從左側導覽列的 **👥（人頭戶名單）** 圖示進入。供「**代多位自然人投票**」的使用者整理戶名對照。

![人頭戶名單](https://raw.githubusercontent.com/kyle2207/Yukuto-releases/main/docs/manual/images/raw-shareholders.png)

### 用途
投票時系統會擷取股東戶名，但若遇**罕用字**（如「珏」）可能與實際姓名不一致。在此登錄你代為操作的股東**正確姓名**，即可作為對照標準，讓「紀念品總覽」依人頭戶正確分組與歸併。

### 操作

| 項目 | 說明 |
|------|------|
| **姓名／備註** | 輸入正確姓名與備註後按「＋ 新增」 |
| 清單欄位 | 姓名、備註、**已歸併變體**（系統自動對應到此姓名的不同寫法）、建立時間 |

> 💡 在「紀念品總覽」中，若某戶名未對應到名單，可用該處的「↪ 歸到」下拉把它歸併到正確的人，系統會記住這個變體，下次自動歸位。

> 📌 **名單不會自動產生**：投票時擷取的戶名只寫進截圖記錄，不會自動加進此名單。「紀念品總覽」會自動依擷取戶名分組（即使沒建名單也看得到各人頭戶）；此名單是讓你**手動正名／歸併變體字**用的。
>
> 🕒 **可事後補建，不影響既有投票**：人頭戶名單純粹是「整理顯示用」的對照。即使你**投完票才新增**，也不會更動任何既有投票（截圖、費用、狀態都不變），只是「紀念品總覽」**下次載入時**會依新名單重新分組／歸併。投票前先建或投票後再補，結果一樣。

## 7. 費用與出貨

### 服務費

### 售出平台費
若委託平台協助轉售，依**售出金額**分段收取：

| 售出金額 | 平台費 |
|---------|-------|
| 未達 NT$ 20 | 免費 |
| NT$ 20 ～ 29 | NT$ 5 |
| NT$ 30 以上 | NT$ 10 |

> 售出平台費於**成功售出後**才收取；售出金額未達門檻者免收。

### 出貨方式
- 紀念品由本平台**統一向股代預約領取**，出貨時間依領貨進度安排；同一委託人之多筆紀念品可**待到齊後集中寄出**，以節省運費。
- 寄送方式：**宅配到府／親取**，可於出貨前選擇並確認運費。
- 預約領貨需**至少 5 個工作天前**提出；如有任何出貨或預約問題，請聯繫管理員協助。

> 親取免運費；實際出貨方式與時間以雙方確認為準。

### 運費
紀念品寄送之運費（物流／郵資）**另計**，依實際出貨方式與件數計收，**不含**於上述服務費與售出平台費內。

> 出貨前會一併確認運費金額再行寄送。

### 付款方式

- 出貨前確認明細及金額，付款後安排出貨；委託本身免預付。

## 9. 常見問題

**Q：需要先安裝 .NET 或其他套件嗎？**
不需要。安裝程式已內含執行環境，唯一的外部需求是 **Google Chrome**（用於驅動投票）。未安裝時首次啟動會提示一鍵安裝。

**Q：電子投票有時間限制嗎？**
有。電子投票平台開放時間為**每日 07:00 ~ 24:00**，請務必在截止日的 24:00 前完成，逾時無法補投。

**Q：紀念品一定要自己一場一場設定嗎？**
不用，兩種方式都可以：
- **委託管理員操作** —— 若保持「**待定**」，需聯繫管理員並委託其處理。
- **自行決定** —— 在「紀念品總覽」為每場選擇處置方式（委託代領／委託販售／贈送／不領取）。

**Q：投票截圖會外洩或佔用我的電腦空間嗎？**
不會。截圖採權限控管、僅存於雲端，約 1 年後自動清除，**不會存放於你的電腦**。

**Q：怎麼更新到最新版？**
不用手動更新。啟動時會自動檢查並提示，一鍵確認即套用（見第 1 章「自動更新」）。

**Q：悠投突然無法正常投票？**
多半是集保平台改版所致，請稍候修正版自動更新；在此期間請改以人工方式於集保平台完成投票。
